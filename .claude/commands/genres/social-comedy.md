# 社会あるあるコメディ コンテンツ作成手順書

## 基本情報

| 項目 | 内容 |
|------|------|
| ジャンルID | `social-comedy` |
| ファイル名 | `social-comedy.md` |
| 対象読者 | 社会人全般（20〜40代） |
| 文字数目標 | 2000〜3000字 |
| 語り口 | 三人称・ナレーター調、または「あるある」語り |

---

## ジャンル定義・コンセプト

歴史上の出来事を「現代の職場・ビジネスシーン」に置き換えて「あるある！」と笑えるコメディに仕立てる。
「稟議書」「1on1ミーティング」「社内チャット」「週次MTG」などの現代的フォーマットを積極的に使う。
笑いの軸は「5000年前から変わらない人間の本質」。教訓はあるあるの延長として自然に忍ばせる。
各エピソードの末尾に「*あるある：〜*」で斜体のまとめを入れる。

---

## 語り手の設定

- **視点**: 冷静なビジネスナレーター。「〜という現象は今も昔も変わらない」
- **口調**: ビジネス文書風と砕けた口語のミックス
- **ツッコミスタイル**: 「*あるある：〜*」の斜体イタリック形式でまとめる
- **感情**: 基本ドライ。ただし最後だけは静かに感動させる

---

## 構成テンプレート

```
【タイトル＋会社名設定】
  「〔株式会社 〜 / 設立〜年〕」の見出しで会社を設立する

【エピソード①「〜あるある」】(400〜500字)
  最初の出来事をビジネスシーン化
  稟議書・面接・承認フローなどのフォーマットを使う
  末尾に「*あるある：〜*」

【エピソード②「〜あるある」】(400〜500字)
  次の出来事
  社内チャット・メール・勤怠管理などのフォーマットを使う
  末尾に「*あるある：〜*」

【エピソード③以降】(繰り返し・400〜500字 × 2〜3)
  歴史上の主要人物ごとにエピソードを作る
  それぞれ異なるビジネスシーンのフォーマットを使う

【まとめ・退職者アンケート】(200〜300字)
  「退職者アンケート」形式で各人物の言葉を集める
  最後だけ感動的な一言を残す
  「——了」で締める
```

## 「あるある」の作り方

1. 歴史的事実の「普遍的な人間行動」を見つける
2. それを現代の職場に置き換える
3. 「*あるある：〜*」で一行にまとめる

**例（良い例）：**
- 「3000人の部下を申請する」→ 「*あるある：部下の人数が多い稟議を出すと「本当に全員必要？」と聞かれる*」
- 「結果を出したら讒言にあう」→ 「*あるある：成果を出すほど嫉妬される*」

**NG：**
- あるあると歴史的事実が対応していない
- あるあるが長すぎる（1〜2行に収める）

## 文体・表現ルール

### ビジネスフォーマットの書き方
```
【稟議書形式】
件名：（タイトル）
目的：（目的）
リソース：（数・規模）

【社内チャット形式】
名前「（メッセージ）」
名前「（返信）」

【メール形式】
件名：（件名）
本文：（内容）
```

### ナレーションのトーン
- 「〜という現象は、今も昔も変わらない」系の普遍化
- 「これが〇〇年前から変わらない職場の真実である」で章を締める
- 冷静・ドライ。感情的にならない

### まとめパートの作り方
- 「退職者アンケート」Q&A形式
- 各人物の「後輩へのメッセージ」を一言で
- 最後の人物だけ「——（回答なし）」や沈黙で余韻を残す

## 必須要素

- [ ] 冒頭に「〔会社名 / 設立年〕」の設定
- [ ] 各エピソードに「*あるある：〜*」（最低1つ）
- [ ] 最低3種類の異なるビジネスフォーマット使用
- [ ] まとめパートに「退職者アンケート」または同等の締め
- [ ] 全体を通して「5000年前から変わらない人間の本質」が見える

---

## 禁止事項

- あるあるが重複する
- 感動シーンが長すぎる（最後の1〜2行に留める）
- ビジネス用語の解説（読者は社会人なので不要）
- 歴史的事実の誇張が行き過ぎて史実が伝わらなくなる

## 冒頭文例

```
## 入社初日〔株式会社 ○○ / 設立○年〕

「では入社手続きを始めます」

（主人公）は書類を受け取り、思わず目を細めた。

**〔雇用条件〕**
- 勤務地：（場所）
- 勤務時間：24時間365日（休日未定）
- 給与：（使命感で）頑張ってください

「……残業代は？」

「（役職）に残業代は発生しません」

「そうですか」

*あるある：神様もサービス残業。*
```
