# ミステリー コンテンツ作成手順書

## 基本情報

| 項目 | 内容 |
|------|------|
| ジャンルID | `mystery` |
| ファイル名 | `mystery.md` |
| 対象読者 | 中学生〜高校生（13〜17歳） |
| 文字数目標 | 2000〜3000字 |
| 語り口 | 一人称（主人公視点） |

---

## ジャンル定義・コンセプト

歴史的事実を「謎」として提示し、主人公が調査・推理しながら真相に近づくミステリー形式。
史実は「証拠」「手がかり」として機能させ、読者が一緒に考えられる構造を作る。
結末は「完全な解決」ではなく「問いを残す」形にして、読者自身の思考を促す。

**核心原則（McKee + クリスティ式）：**
「謎の提示→調査→誤った仮説→新証拠→解決」の5段構造。
各章末は「問い」か「新情報」で終わらせ、「答え」で終わらせない。

---

## 語り手・主人公の設定

- **主人公像**: 好奇心旺盛な十代。探偵役または歴史好きの学生
- **視点**: 一人称（「俺」「私」どちらでも可・原典の雰囲気に合わせる）
- **口調**: 現代語。思考が明瞭で、観察描写が細かい
- **重要設定**: 「観察過多」な人物。普通の人が見逃す細部に気づく語り手として設定する。これが読者に「発見の快感」を提供する

---

## フックの技法（冒頭で読者を引き込む）

1. **解答を先に出してから謎を提示する（逆順フック）**：
   - 例：「犯人は最初からわかっていた。問題は、証明できないことだった」
   - 読者に「なぜ？」「どうやって？」という問いを最初に植える
2. **In Medias Res（物語中盤から開始）**：謎・異変・犯罪をすでに発生している状態から始める。説明より先に「おかしい」を見せる
3. **「見ている場所が違う」冒頭**：
   - 例：「他の全員が古文書に目を向けていた。でも私だけは、ページの端に何かが書き込まれているのに気づいていた」
   - 主人公の観察眼をフックとして使う

---

## ページをめくらせる仕掛け

- **ドラマティック・アイロニー**：読者が知っていて主人公が知らない情報（または逆）を常に作る。この非対称が緊張を生む
- **情報の「2出し1留保」ルール**：毎シーンで謎を2つ解消して1つ新たに作る。情報の過剰提供は飽きを生む
- **中盤の「偽りの解決」**：物語の60%地点で「犯人がわかった」と見せかけてから覆す。最もページをめくらせる構造
- **タイムリミットを具体的な数字で設定**：「72時間以内に手がかりを見つけなければ、この謎は永遠に消える」——時計の針は最強の緊張装置

---

## 構成テンプレート

```
【第一章：謎の提示】(300〜400字)
  「解答を先に出してから謎を提示」で始める（逆順フック）。
  主人公が不思議な物（古文書・記録・場所）に出会う。
  章末：「謎の問い」を残す。絶対に答えを置かない。

【第二章：調査開始・手がかり①】(400〜500字)
  手がかり①を発見・分析する。
  歴史的事実を「証拠」として提示。
  「2出し1留保」：情報2つを提示し、1つは疑問として残す。
  章末：「しかしこれだけでは足りない」で保留。

【第三章：手がかり②と誤った仮説】(400〜500字)
  別の角度から手がかり②を発見。
  主人公が仮説を立てる——しかしこれは「偽りの解決」に向かう。
  読者には「この仮説はおかしい」と気づかせる伏線を1つ埋める（ドラマティック・アイロニー）。

【第四章：誤った仮説の崩壊・手がかり③】(300〜400字)
  決定的な手がかり③が偽りの解決を崩す。
  この展開が最大のページターン装置。

【第五章：真相と残る問い】(300〜400字)
  主人公なりの結論を出す。
  ただし「全部の答えではない」として1つの問いを残す。
  読者への問いかけで締める。
```

## 文体・表現ルール

### ミステリー演出
- 斜体（`*〜*`）で主人公の内なる推理・思考を表現する
- 「おかしい」「違和感」「なぜ」を繰り返して緊張を積み上げる
- 章末は必ず「問い」か「不穏な新情報」で終わらせる

### ペーシング（スティーヴン・キング式）
- **緊張シーンは1文=1動作**：「扉が開く。足音がする。電気が消えた。」——短文の連打が心拍を上げる
- **調査シーンは長文で思考を流す**：主人公が考える時間を長文で作る。読者が一緒に推理できる余白
- **章末は短文で締める**：最後の1〜2文を短くして「余韻」と「引き」を同時に作る

### 情報の出し方
- 手がかりは1章に1つ。まとめて出さない
- 「一部の歴史家は〜と指摘している」など複数の解釈があることを示す
- 神話・伝説部分は「真偽不明だが……」と明示する（信頼性の調整）

## 必須要素

- [ ] 冒頭に「逆順フック」（解答先出し、またはIn Medias Res）
- [ ] 手がかりが最低3つ（1章1つずつ）
- [ ] 各章の末尾に「問い」か「新情報」（答えを置かない）
- [ ] 中盤に「偽りの解決」とその崩壊
- [ ] 「ドラマティック・アイロニー」の仕掛けが1か所
- [ ] 「完全解決せず問いを残す」結末

---

## 禁止事項

- 冒頭を説明から始める（謎・異変からIn Medias Resで始める）
- 謎が最初から全部わかる展開
- 手がかりを1章にまとめて出す（1章1手がかりを守る）
- 章末を「答え」で締める（必ず「問い」か「新情報」で終わらせる）
- 主人公が調べもせず即座に答えを出す

## 冒頭文例

```
結論から言う。

あの文字が「偽物」だということは、最初から分かっていた。

問題は——なぜ5000年前の人間が、わざわざ偽物を残したのかということだ。

図書館の片隅で、僕は一冊の古びた本を見つけた。
ページを開くと、インクの滲んだ文字が並んでいる。

*（これは……普通の写本じゃない）*

そこでページが破れていた。

「……なんでここだけ」

その疑問が、すべての始まりだった。
```
