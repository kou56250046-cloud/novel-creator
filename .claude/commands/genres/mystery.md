# ミステリー コンテンツ作成手順書

## 基本情報

| 項目 | 内容 |
|------|------|
| ジャンルID | `mystery` |
| ファイル名 | `mystery.md` |
| 対象読者 | 中学生〜高校生（13〜17歳） |
| 文字数目標 | 2000〜3000字 |
| 語り口 | 一人称（主人公視点） |

---

## ジャンル定義・コンセプト

歴史的事実を「謎」として提示し、主人公が調査・推理しながら真相に近づくミステリー形式。
史実は「証拠」「手がかり」として機能させ、読者が一緒に考えられる構造を作る。
結末は「完全な解決」ではなく「問いを残す」形にして、読者自身の思考を促す。

---

## 語り手・主人公の設定

- **主人公像**: 好奇心旺盛な十代。探偵役または歴史好きの学生
- **視点**: 一人称（「俺」「私」どちらでも可・原典の雰囲気に合わせる）
- **口調**: 現代語。思考が明瞭で、観察描写が細かい
- **感情**: 謎を前にすると冷静になる。ただし核心に触れると揺れる

---

## 構成テンプレート

```
【第一章：謎の提示】(300〜400字)
  主人公が不思議な物（古文書・記録・場所）に出会う
  「おかしい」「普通と違う」という違和感を植える
  章末に「謎の問い」を残す

【第二章：調査開始・最初の手がかり】(400〜500字)
  手がかり①を発見・分析する
  歴史的事実を「証拠」として提示する
  新たな疑問が生まれる

【第三章：手がかり②】(400〜500字)
  別の角度から手がかり②を発見
  手がかり①と②が矛盾しているように見える、または補完する

【第四章：手がかり③と仮説】(任意・300〜400字)
  決定的な手がかり③
  主人公が仮説を立てる

【第五章：真相と残る問い】(300〜400字)
  主人公なりの結論を出す
  ただし「全部の答えではない」として問いを残す
  読者への問いかけで締める
```

---

## 文体・表現ルール

### ミステリー演出
- 「おかしい」「違和感」「なぜ」を繰り返し使って緊張感を高める
- 斜体（`*〜*`）で主人公の内なる推理・思考を表現する
- 章末は必ず「問い」か「新たな謎」で終わらせる

### 情報の出し方
- 手がかりは一度に全部出さない（1章に1つ）
- 手がかりを提示したあと、すぐに「しかしこれだけでは足りない」と保留する
- 結論は段階的に：「疑問→仮説→検証→答え（ただし不完全）」

### 歴史的事実の使い方
- 史実を「証拠」と「状況証拠」に分けて整理して提示する
- 「一部の歴史家は〜と指摘している」など複数の解釈があることを示す
- 神話・伝説部分は「真偽不明だが……」と明示する

### 文体
- 観察描写は短く正確に（長い形容詞連鎖は避ける）
- 思考の描写は長めにして「読者が一緒に考えられる」余白を作る
- 会話は少なめ。地の文中心

---

## 必須要素

- [ ] 冒頭で「謎のある物・場所・情報」と出会うシーン
- [ ] 手がかりが最低3つ
- [ ] 各手がかりに対応する史実（または伝承）の引用
- [ ] 主人公が仮説を立てる場面
- [ ] 「完全には解決しない」結末と読者への問いかけ

---

## 禁止事項

- 謎が最初から全部わかる展開（謎は最後まで少し残す）
- 史実の断定的な否定（「これは嘘だ」ではなく「疑問が残る」に留める）
- 説明的な歴史解説の羅列（必ず物語の流れの中に組み込む）
- 主人公が調べもせず即座に答えを出す

## 冒頭文例

```
図書館の片隅、僕は一冊の古びた本を見つけた。

タイトルは『〜の秘密』。

ページを開くと、インクの滲んだ文字が並んでいる。

*「この民には、天が定めた使命がある。だが、その使命を知る者は——」*

そこでページが破れていた。

「……なんだこれ」
```
