---

## ジャンル定義・コンセプト

歴史上の出来事を「現代の職場・ビジネスシーン」に置き換えて「あるある！」と笑えるコメディに仕立てる。
「稟議書」「1on1ミーティング」「社内チャット」「週次MTG」などの現代的フォーマットを積極的に使う。
笑いの軸は「5000年前から変わらない人間の本質」。

**核心原則（ホラティウス的サタイア＋The Onion式）：**
読者が「自分もやっていること」を認めさせてから笑わせる。自己認識を先に起こさせることで、笑いがカタルシスになる。
「あるある→解決せず→さらに悪化→諦め」の循環が社会諷刺の本質。

---

## 語り手の設定

- **視点**: 達観した当事者。怒りも悲しみもなく、ただ淡々と不条理を観察・報告する
- **口調**: ビジネス文書風と砕けた口語のミックス
- **重要技法**: サブテキストを機能させる。「課長はその場でとても喜んでいた」という文が実は「課長の無知が露呈した」を意味する構造。読者が「読めた」と感じる快感が次を読む動機になる

---

## フックの技法（冒頭で読者を引き込む）

1. **「真顔の前提」から始める（The Onion式）**：
   - 例：「紀元前2333年、株式会社・白頭山では本日、新たな事業部長が空から降臨した。採用経緯は不明」
   - 現実にありそうな「おかしさ」を真面目に語ることで即座に笑いを作る
2. **読者が「自分のことだ」と思う設定を冒頭に置く**：
   - 例：「〔勤務地：白頭山頂。交通手段：神の力。休日：未定〕」
   - 具体的な職種・状況に読者の経験を接続する
3. **具体的な数字で「あるある」を始める**：「部下3000人の稟議申請。担当者は一人」——数字の具体性がリアリティを上げ笑いを強める

---

## ページをめくらせる仕掛け

- **「1段落=1観察」のリズム**：短い段落で次々と「あるある」を積み重ねる。読者に「そうそう！次は？」という軽いリズムを作る
- **視点の転換を中盤に入れる**：同じ状況を上司・部下・取引先の視点で描く。社会の歪みが多角的に見えて笑いが深まる
- **オチは「解決しない」か「さらに悪化する」**：解決してはいけない。「まだ続いている」という余韻が社会諷刺の生命力
- **サブテキスト（行間の意味）を埋める**：表向きの建て前と本音のギャップを毎エピソードに入れる

## 構成テンプレート

```
【タイトル＋会社名設定】
  「〔株式会社 〜 / 設立〜年〕」の見出しで会社を設立する。
  雇用条件を書類形式で提示——すでにここが笑いポイント。

【エピソード①「〜あるある」】(400〜500字)
  状況提示（真顔） → 問題発生 → 「*あるある：〜*」
  ビジネスフォーマット（稟議書・チャットなど）を使う。
  「1段落=1観察」リズムで積み重ねる。

【エピソード②「〜あるある」】(400〜500字)
  視点を変える（上司→部下、または別の登場人物視点）。
  サブテキスト仕掛け：表向きの発言＋本音の乖離を入れる。

【エピソード③以降】(400〜500字 × 2〜3)
  各エピソードで異なるフォーマット・異なる人物を使う。
  エスカレーション：前のエピソードより「さらに不条理」にする。
  末尾に「*あるある：〜*」

【まとめ・退職者アンケート】(200〜300字)
  「退職者アンケート」形式で各人物の回答を集める。
  最後だけ「——（回答なし）」か短い感動の一言で余韻。
  「——了」で締める。
```

## 使用できるフォーマット一覧

| フォーマット | ブラック的な使い方 |
|-------------|----------------|
| 稟議書 | 申請内容が正気じゃない条件を「普通に」列挙 |
| 社内チャット | 深夜・緊急・片方が既読無視のやり取り |
| 週次MTG議事録 | 参加者が多すぎる・意味のないKPIが並ぶ |
| 採用面接 | 明らかにおかしい雇用条件を淡々と提示 |
| 業績評価メール | 偉業を達成したのに「もう少し頑張れ」 |
| 退職者アンケート | 「天命を全うしたので」「戦死のため」 |
| 表彰メール | 表彰直後に重大ミッションを命じられる |

## 「あるある」の作り方

1. 歴史的事実の「普遍的な人間行動」を見つける
2. それを現代の職場フォーマットに置き換える
3. 具体的な数字・固有名詞・データで「リアリティ」を足す
4. 「*あるある：〜*」で一行にまとめる（1〜2行厳守）

**良い例：**
- 「3000人の部下を申請する稟議」→ `*あるある：部下の人数が多い稟議を出すと「本当に全員必要？」と聞かれる*`
- 「全勝したのに謹慎処分」→ `*あるある：結果を出すほど嫉妬される*`

## 必須要素

- [ ] 冒頭に「〔会社名 / 設立年〕」の設定
- [ ] 「真顔の前提」（おかしい状況を真面目に提示）で始める
- [ ] 各エピソードに「*あるある：〜*」（最低1つ）
- [ ] 最低3種類の異なるビジネスフォーマット使用
- [ ] 視点の転換が1か所
- [ ] サブテキスト仕掛けが1か所（建て前と本音のギャップ）
- [ ] オチが「解決しない」または「悪化する」
- [ ] 退職者アンケートまたは同等の締め

---

## 禁止事項

- あるあるが重複する
- 問題が解決してしまう（社会諷刺のオチは「まだ続いている」）
- 感動シーンが長すぎる（最後の1〜2行に留める）
- 歴史的事実の誇張が行き過ぎて史実が伝わらなくなる

## 冒頭文例

```
## 入社初日〔株式会社 白頭山 / 設立 紀元前2333年〕

「では入社手続きを始めます」

桓雄は書類を受け取り、思わず目を細めた。

**〔雇用条件〕**
- 勤務地：白頭山山頂（交通費支給なし・交通手段なし）
- 勤務時間：24時間365日（休日「随時検討」）
- 給与：「天命」で頑張ってください
- 部下：3000名（採用稟議：天帝承認済み）

「……残業代は？」

「神の使命に残業代は発生しません」

「そうですか」

*あるある：「やりがい」という言葉が出た瞬間、給与の話は終わる。*
```
