# ラノベ系ストーリー コンテンツ作成手順書

## 基本情報

| 項目 | 内容 |
|------|------|
| ジャンルID | `lightnovel` |
| ファイル名 | `lightnovel.md` |
| 対象読者 | 中学生〜大学生（13〜22歳） |
| 文字数目標 | 2500〜4000字 |
| 語り口 | 一人称（主人公視点）または主人公密着三人称 |

---

## ジャンル定義・コンセプト

歴史上の人物を「個性の強いキャラクター」として描き、ドラマチックな演出・熱い展開・感情の起伏で読ませるラノベ風物語。
「プロローグ→各話→エピローグ」の章立て構成で、各キャラクターの見せ場（名シーン）を作る。
テーマは「使命・覚悟・成長」。感情を大きく動かす「熱いシーン」を少なくとも2か所作る。

**核心原則（なろう系＋McKee）：**
冒頭2000字以内で「主人公が動いている状態」を見せる。説明から始めない。
感情の振れ幅「緊張→笑い→切なさ→高揚感」を1話の中に設計する。

---

## 語り手・主人公の設定

- **主人公像**: 選ばれし運命を持つ者。強さとともに迷いや感情を持つ
- **視点**: 一人称（「俺」）または主人公に密着した三人称
- **口調**: 現代的かつ文学的。「——そうだ」「——いや、違う」の独白スタイル
- **重要設定**: 序盤に必ず「弱さ」を見せる（Save the Cat原則）。強さだけを見せると読者が応援する動機を失う

---

## フックの技法（冒頭で読者を引き込む）

1. **主人公が「動いている状態」で開始**：
   - 例：「剣が砕けた。その瞬間、俺は三千年の呪いが終わったことを知った」
   - 説明より先に「何かが起きた直後」から始める
2. **予告的引用（ラノベの定番技法）**：物語後半の印象的なセリフを冒頭に置く：
   - 例：「——後にこれが、すべての始まりだったと彼女は言った」
   - 読者に「どうしてこうなったのか」を最初から問わせる
3. **「この世界のルール違反」を1ページ目に見せる**：異常な状況・常識外の出来事を冒頭に置く。世界観説明は後回しにする

---

## ページをめくらせる仕掛け

- **章末クリフハンガーは必ず「新情報の投下」か「状況の急転」**：既存情報の確認では終わらせない。「——そのとき、使者が来た。送り主：死んだはずの〇〇」
- **感情の振れ幅設計**：「緊張→笑い→切なさ→高揚感」のサイクルを1話内に設計する。同じトーンが続くと飽きる
- **見せ場は85%地点に置く**：最後の15%で余韻を作る。名セリフ・名場面がここにくる
- **ライバル・敵キャラを「理解できる論理」で動かす**：敵の主張に一定の正しさを持たせると知的緊張が生まれ読者を引きつける

---

## 構成テンプレート

```
【プロローグ】(200〜300字)
  「予告的引用」か「物語後半の一場面」で始める。
  「——これはその始まりの物語だ」的な導入。
  読者に「なぜこうなったのか」という問いを植える。

【第一話〜第N話】(各500〜700字)
  各話に一人のキャラクターを割り当てる。
  各話の設計：
    - 「動いている状態」からの開始（説明なし・100字以内で情況提示）
    - キャラクターの弱さ→課題→葛藤（200字）
    - 行動・決断（200字）：迷いを経て決断する
    - 名シーン・名台詞（100字）：85%地点に置く
    - 章末：「問い」か「新情報」（答えで終わらせない）
  話数は原典の主要人物数に合わせる（3〜5話）

【エピローグ】(200〜300字)
  物語全体を俯瞰する締め。
  主人公の成長・気づきを「行動」で表現（説明しない）。
  「——END」または「——了」で締める。
```

## 文体・表現ルール

### 演出テクニック
- ダッシュ（——）を多用して余韻を作る（「決めた——」「静かだった——」）
- **体言止め連打でバトル・決断シーン**：「斬る。受ける。踏み込む。——決まった。」
- **内面独白は「——」を使った中断文**：「決めた——いや、違う。俺はまだ——」——思考の揺れが読者を同期させる

### ペーシング
- **緊張場面は短文連打**：1文=1動作。5文以内で場面を転換する
- **感情的シーンは1文を長く伸ばして「流す」**：「彼女が泣いているのを見て、俺はようやく——ずっと気づかないふりをしてきた自分の本当の気持ちに——初めて、向き合った」
- **地の文を会話と同じ口語レベルで書く**：「彼は言った」ではなく「俺は言った。なんかかっこつけながら」——キャラクターの個性が地の文に滲む

### キャラクター描写
- 外見は最小限（特徴的な描写1つのみ）
- 脇役の反応で主人公の「すごさ」を際立てる（主人公自身に語らせない）
- 弱さは行動で見せる（「〇〇が怖い」と言わせず、怖い場面でのリアクションを見せる）

---

## 禁止事項

- 冒頭を「設定説明」から始める（動いている状態から始める）
- キャラクターが完璧すぎる（弱さを必ず持たせる）
- 同じ感情トーンが続く（緊張→笑い→切なさのサイクルを守る）
- 章末を「答え」で終わらせる（必ず「問い」か「新情報」）
- 長い説明的な地の文（場面転換は1行の情景描写で行う）

## 冒頭文例

```
「——お前は、すべてを知らなければならない」

それが俺の最後の記憶だ。

白い光。広大な大地。そびえる山。そして——三千人の部下。

目が覚めたとき、俺は空の上に立っていた。

（……なんだこれ）

混乱するな、と自分に言い聞かせた。でも、足が震えている。それが、この旅の最初の「弱さ」だった。

「桓雄様！ご無事でございますか！！」

——俺は、まだ生きていた。問題は、ここが「どこ」かだった。
```
